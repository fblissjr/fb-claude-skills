# writing

*Last updated: 2026-06-27*

Writing skills for clear, accessible prose. The first skill applies GOV.UK / Government Digital Service (GDS) house style — plain English, active voice, front-loaded content, sentence case, and no bold or italics for emphasis.

## Installation

```bash
/plugin install writing@fb-claude-skills
```

## Skills

| Skill | Trigger | Description |
|-------|---------|-------------|
| `govuk-style` | "GOV.UK style", "plain English", "rewrite in GDS style", "make this clearer" | Write and edit reports, guidance, and summaries in GOV.UK house style |

## Invocation

```
/writing:govuk-style
```

Or trigger automatically by asking to write or rewrite a report, research write-up, guidance, or summary in plain, accessible English.

## Background

Based on the [GOV.UK style guide](https://www.gov.uk/guidance/style-guide) and GDS content design principles. The goal is to open the content up so anyone understands it on first read, without losing substance, nuance, or precision.

Adapted from a skill shared by [@fofr](https://twitter.com/fofr).
